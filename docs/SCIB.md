# СКИБ для проекта ORVD System

Система Контроля и Информационного Блокирования (СКИБ) для управления тестированием и безопасностью компонентов ORVD-системы.

## Планирование

### Компоненты системы

| Компонент        | Описание                                           | Порт / Настройка   |
| ---------------- | -------------------------------------------------- | ------------------ |
| ORVD Gateway     | Интерфейс для отправки команд и регистрации дронов | HEALTH_PORT=8080   |
| ORVD Component   | Основной компонент обработки миссий и сообщений    | MQTT/Kafka брокеры |


### Задачи и цели

| ID задачи | Компонент      | Цель                        | Риски                         | Статус         |
| --------- | -------------- | --------------------------- | ----------------------------- | -------------- |
| T1        | Gateway        | Регистрация дрона           | Брокер недоступен, DNS ошибка | В планировании |
| T2        | ORVD Component | Авторизация миссии          | Некорректные сообщения        | В планировании |
| T3        | Kafka          | Проверка доставки сообщений | DNS/порт недоступны           | В планировании |
| T4        | Component      | Сбор истории событий        | Потеря сообщений              | В планировании |


## Реализация

### Инструкция по запуску

- Создать и проверить Docker-сеть:

```bash
docker network create drones_net
```

- запустить брокеры через docker compose

```bash
docker compose -f docker-broker.yml up -d --build
```

- запустить компоненты системы через подготовленный Compose:

```bash
docker compose -f docker-compose.yml --profile kafka --profile mqtt up -d --build
```

- проверить статус контейнеров

```bash
docker ps
```

### Сценарий тестирования

| ID задачи | Шаг реализации                            | Ожидаемый результат              | Лог / Доказательство |
| --------- | ----------------------------------------- | -------------------------------- | -------------------- |
| T1        | Отправить REGISTER_DRONE через gateway    | Ответ со статусом `"registered"` | Логи Kafka и pytest  |
| T2        | Отправить AUTHORIZE_MISSION через gateway | Ответ со статусом `"authorized"` | Логи сообщений       |
| T3        | Проверить доставку сообщений в Kafka      | Все сообщения доставлены         | Логи Kafka + метрики |
| T4        | Проверить историю событий компонента      | В ответе есть ключ `"history"`   | Логи компонента      |


### Ожидаемые проблемы и решения

| Проблема                    | Возможная причина                             | Решение                                               |
| --------------------------- | --------------------------------------------- | ----------------------------------------------------- |
| DNS lookup failed для Kafka | Контейнеры не в одной сети                    | Проверить `networks` в Docker Compose                 |
| Ошибка публикации в Kafka   | Broker не поднят                              | Запустить `docker compose -f docker-broker.yml up -d` |
| Ответы None в pytest        | Компоненты не стартовали или misconfiguration | Проверить переменные окружения и профили Compose      |
